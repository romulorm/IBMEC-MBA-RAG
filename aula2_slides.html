Por que Chunking é a Decisão mais CríticaAula 2

1/6

O chunk é a menor unidade de informação recuperável. A qualidade da resposta é limitada pela qualidade dos chunks.

  • ⚠️ Chunk muito pequeno: perde contexto, não responde perguntas complexas
  • ⚠️ Chunk muito grande: ruído, degrada Context Precision no RAGAS
  • ⚠️ Corte no meio de sentença: embedding incoerente = retrieval incorreto
  • ✅ Boa estratégia = unidade semântica coerente + tamanho calibrado ao domínio

Estratégia 1 — Fixed-Size ChunkingAula 2

2/6

Divide o texto em janelas fixas de N tokens com sobreposição (overlap) para não perder contexto entre chunks.

  • chunk_size: 512 tokens (padrão jurídico — cabe 1-2 parágrafos)
  • chunk_overlap: 64 tokens (10-15%) — garante continuidade de argumentos
  • Vantagem: simples, previsível, funciona bem para textos uniformes
  • Desvantagem: corta sentenças, desconhece estrutura do documento
Python 3.11+
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=512,
    chunk_overlap=64,
    length_function=len,
    separators=["\n\n", "\n", ". ", " "]
)

chunks = splitter.split_text(acórdão_texto)
print(f"Total chunks: {len(chunks)}")
print(f"Chunk 0 preview: {chunks[0][:200]}")

Estratégia 2 — Semantic ChunkingAula 2

3/6

Detecta quebras semânticas naturais usando similaridade de embeddings entre sentenças consecutivas.

  • Calcula similaridade coseno entre embeddings de sentenças adjacentes
  • Quebra onde a similaridade cai abaixo de um threshold (ex: percentile 95)
  • Preserva a coerência temática — ideal para acórdãos com seções distintas
  • Custo: N embeddings por documento na ingestão (mais lento que fixed-size)
Python 3.11+
from langchain_experimental.text_splitter import SemanticChunker
from langchain_community.embeddings import HuggingFaceBgeEmbeddings

embeddings = HuggingFaceBgeEmbeddings(
    model_name="BAAI/bge-m3"
)

splitter = SemanticChunker(
    embeddings,
    breakpoint_threshold_type="percentile",
    breakpoint_threshold_amount=95
)

chunks = splitter.split_text(acórdão_texto)
print(f"Chunks semânticos: {len(chunks)}")

Estratégia 3 — Sentence-Window ChunkingAula 2

4/6

Indexa frases individuais mas recupera janelas maiores (contexto) ao fazer o retrieval.

  • Índice: 1 sentença por chunk (preciso para retrieval)
  • Recuperação: entrega janela de ±2 sentenças (contexto para o LLM)
  • Desacopla o que é indexado do que é gerado — implementado em LangChain + NLTK (Python puro)
  • Ideal para documentos onde a resposta está em trechos específicos mas precisa de contexto
Python 3.11+
from langchain.schema import Document
from nltk.tokenize import sent_tokenize

def sentence_window_chunking(texto, window_size=2):
    sentencas = sent_tokenize(texto)
    docs = []
    for i, sent in enumerate(sentencas):
        ini, fim = max(0, i-window_size), i+window_size+1
        janela = " ".join(sentencas[ini:fim])
        docs.append(Document(
            page_content=sent,                  # sentença indexada
            metadata={"window": janela}    # contexto para o LLM
        ))
    return docs

docs = sentence_window_chunking(acórdão_texto, window_size=2)
print(f"Documents criados: {len(docs)}")

Docling — Ingestão InteligenteAula 2

5/6
Docling (IBM Research) converte PDFs complexos em DoclingDocument estruturado, preservando tabelas, layout e hierarquia.
  • Extrai texto, tabelas, figuras, cabeçalhos e rodapés com fidelidade
  • Detecta estrutura hierárquica: Título → Seção → Parágrafo
  • Suporte a Header-Based Chunking — respeita a estrutura do acórdão
  • Saída: DoclingDocument → LangChain Documents → OpenSearch
Python 3.11+
from docling.document_converter import DocumentConverter
from docling.chunking import HybridChunker

converter = DocumentConverter()
result = converter.convert("acordao_stf.pdf")
doc = result.document

# Header-based chunking via Docling
chunker = HybridChunker(tokenizer="BAAI/bge-m3")
chunk_iter = chunker.chunk(dl_doc=doc)
chunks = list(chunk_iter)
print(f"Chunks estruturados: {len(chunks)}")
print(chunks[0].text[:300])

Pipeline Naive RAG CompletoAula 2

6/6
O Naive RAG é o baseline: ingere → indexa → recupera → gera. Simples e poderoso.
  • 1. Ingestão: Docling converte PDF → chunks de texto
  • 2. Indexação: BGE-M3 gera embeddings → OpenSearch kNN index
  • 3. Retrieval: query embedding → busca top-K por similaridade coseno
  • 4. Geração: chunks + query → Ollama (llama3.2:3b — Aula 1) → resposta fundamentada
