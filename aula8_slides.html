Limitações do RAG ConvencionalAula 8

1/6

O RAG linear (recuperar → gerar) não tem mecanismos de verificação ou correção.

  • Recuperação indiscriminada: busca sempre, mesmo quando a resposta é conhecimento geral
  • Sem verificação de relevância: usa documentos independentemente de sua pertinência
  • Sem autocorreção: se o retrieval falha, gera resposta imprecisa sem detectar
  • Em Direito: erro factual pode ter consequências reais — precisamos de verificação

Self-RAG — Os 4 Tokens de ControleAula 8

2/6
Self-RAG treina o modelo para emitir tokens especiais que controlam quando e como recuperar.
  • [Retrieve] yes/no — decide se precisa buscar documentos
  • [ISREL] yes/no — avalia se o documento recuperado é relevante para a query
  • [ISSUP] yes/no — avalia se a geração é suportada pelo documento
  • [ISUSE] A/B/C/D — avalia a utilidade geral da resposta gerada

CRAG — Corrective RAGAula 8

3/6

CRAG adiciona um avaliador explícito e fallback para web search quando o retrieval local falha.

  • Avaliador: LLM classifica cada chunk como CORRETO / INCORRETO / AMBÍGUO
  • Threshold CORRETO: usa chunks diretamente na geração
  • Threshold AMBÍGUO ou INCORRETO: aciona web search via Tavily como fallback
  • Fusão: combina resultados locais (verificados) com resultados da web
Python 3.11+
# Avaliador CRAG
GRADER_PROMPT = """
Avalie se o documento é relevante para responder a pergunta.

Pergunta: {query}
Documento: {doc}

Resposta (apenas "correto", "incorreto" ou "ambíguo"):
"""

def grade_document(query: str, doc: str, llm) -> str:
    return llm.generate(
        GRADER_PROMPT.format(query=query, doc=doc),
        temperature=0,
        max_tokens=10
    ).strip().lower()

# Roteamento
def crag_route(grades: list[str]) -> str:
    if all(g == "correto" for g in grades):
        return "generate"      # retrieval OK
    elif any(g == "correto" for g in grades):
        return "filter_generate"  # filtrar e gerar
    else:
        return "web_search"    # fallback Tavily

LangGraph — Orquestração de GrafosAula 8

4/6

LangGraph permite construir pipelines como grafos com nós, arestas e ciclos condicionais.

  • StateGraph: define o estado compartilhado entre todos os nós
  • Nós (nodes): funções Python que recebem e retornam o estado
  • Arestas condicionais: direcionam o fluxo baseado no estado atual
  • Ciclos: permite loops de correção — fundamental para CRAG e Self-RAG
Python 3.11+
from langgraph.graph import StateGraph, END
from typing import TypedDict

class CRAGState(TypedDict):
    query: str
    documents: list
    generation: str
    needs_web_search: bool

# Definir grafo
workflow = StateGraph(CRAGState)

# Adicionar nós
workflow.add_node("retrieve", retrieve_node)
workflow.add_node("grade", grade_node)
workflow.add_node("web_search", web_search_node)
workflow.add_node("generate", generate_node)

# Arestas condicionais
workflow.add_conditional_edges(
    "grade",
    crag_route,  # função de roteamento
    {
        "generate": "generate",
        "web_search": "web_search",
        "filter_generate": "generate"
    }
)
app = workflow.compile()

Tavily — Web Search como FallbackAula 8

5/6

Tavily é uma API de busca otimizada para LLMs, retornando trechos limpos e relevantes.

  • Retorna snippets já processados — sem necessidade de scraping adicional
  • search_depth='advanced': analisa páginas em profundidade para maior precisão
  • max_results=3-5: controla quantidade de fontes externas
  • Caso de uso CRAG: STF publicou novo acórdão após o knowledge cutoff do LLM
Python 3.11+
from tavily import TavilyClient

client = TavilyClient(api_key=os.environ["TAVILY_API_KEY"])

def web_search_fallback(query: str) -> list[str]:
    results = client.search(
        query=f"{query} jurisprudência STF STJ",
        search_depth="advanced",
        max_results=3,
        include_domains=["stf.jus.br", "stj.jus.br"]
    )
    return [
        f"[Web] {r['title']}\n{r['content']}"
        for r in results["results"]
    ]

Comparativo: Self-RAG vs CRAG vs Advanced RAGAula 8

6/6

Cada técnica tem seu nicho de aplicação no contexto jurídico.

  • Advanced RAG: pipeline pré-definido, sem loops — mais rápido, menos adaptativo
  • Self-RAG: recuperação seletiva sob demanda — reduz latência em queries simples
  • CRAG: fallback para web search — essencial quando corpus local está desatualizado
  • LangGraph: orquestra qualquer um dos três com rastreabilidade completa
