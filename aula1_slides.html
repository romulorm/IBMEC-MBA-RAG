Por que RAG? O Problema dos LLMsAula 1

1/13

LLMs são poderosos mas têm limitações críticas em contextos jurídicos e de segurança pública.

  • 🔴 Hallucination — inventa fatos com confiança (ex: cita artigo inexistente do CPP)
  • 🔴 Knowledge cutoff — não conhece legislação ou jurisprudência recente
  • 🔴 Sem fonte verificável — impossível auditar a origem das respostas
  • 🔴 Contexto de janela limitado — não processa todo o acervo documental
  • ✅ Solução: RAG injeta contexto recuperado de fontes verificáveis em tempo real

Tópico 1 — NLP: FundamentosAula 1

2/13

Processamento de Linguagem Natural (NLP) é o campo da IA que permite às máquinas compreender e gerar linguagem humana.

  • Bag-of-Words: representação esparsa, ignora ordem e semântica
  • TF-IDF: pesa termos pela frequência vs raridade no corpus
  • BM25: variação probabilística do TF-IDF, base do Elasticsearch/OpenSearch
  • Problema central: representar o significado do texto, não apenas as palavras
Python 3.11+
from sklearn.feature_extraction.text import TfidfVectorizer

corpus = [
    "o réu foi preso em flagrante",
    "habeas corpus impetrado pelo advogado",
]
vec = TfidfVectorizer()
X = vec.fit_transform(corpus)
print(X.toarray())  # matriz esparsa

Tópico 2 — Embeddings EstáticosAula 1

3/13

Word2Vec, GloVe e fastText representam palavras como vetores densos no espaço semântico.

  • Word2Vec (2013): rede neural shallow, aprende de co-ocorrências — 'rei' - 'homem' + 'mulher' ≈ 'rainha'
  • GloVe: minimiza diferença entre log de co-ocorrências globais
  • fastText: decompõe palavras em subpalavras — lida com OOV
  • Limitação crítica: uma palavra = um vetor fixo, sem contexto (banco = instituição OU assento?)
Python 3.11+
from gensim.models import Word2Vec

# Treinar embeddings jurídicos
sentences = [
    ["réu", "habeas", "corpus", "STF"],
    ["prisão", "preventiva", "decreto"],
]
model = Word2Vec(sentences, vector_size=100, window=5)
print(model.wv.most_similar("réu", topn=3))

Tópico 3 — Embeddings Contextuais & BGE-M3Aula 1

4/13

BERT e seus descendentes geram vetores dependentes do contexto completo da sentença.

  • BERT (2018): Transformer bidirecional — 'banco' tem vetor diferente em cada contexto
  • Sentence-Transformers: fine-tuning para similaridade entre frases (STS)
  • BGE-M3 (BAAI): estado-da-arte multilíngue, suporta 100+ idiomas e português jurídico
  • dim=1024, max_length=8192 — ideal para documentos jurídicos longos
Python 3.11+
from FlagEmbedding import BGEM3FlagModel

model = BGEM3FlagModel("BAAI/bge-m3", use_fp16=True)

queries = ["prisão preventiva requisitos"]
docs = ["Art. 312 CPP: A prisão preventiva poderá ser decretada..."]

q_emb = model.encode(queries, batch_size=4)
d_emb = model.encode(docs, batch_size=4)

# Similaridade coseno
import numpy as np
score = np.dot(q_emb[0], d_emb[0])
print(f"Similaridade: {score:.4f}")

O mecanismo de Self-Attention é a base de todos os LLMs modernos (GPT, Llama, BERT).

  • Self-Attention: cada token atende a todos os outros — captura dependências de longo alcance
  • Multi-Head Attention: múltiplas projeções paralelas de Q, K, V
  • Positional Encoding: injeta informação de posição (sem recorrência)
  • Encoder: compreensão (BERT) | Decoder: geração (GPT, Llama) | Encoder-Decoder: tradução (T5)

Tópico 5 — LLMs: GPT, Llama e ParâmetrosAula 1

6/13

Grandes Modelos de Linguagem são Transformers decoder-only treinados em trilhões de tokens.

  • Llama 3.1 8B: open-source, roda localmente via vLLM, LGPL
  • temperature: controla aleatoriedade (0 = determinístico, 1 = criativo)

Tópico 6 — O que é RAG?Aula 1

7/13
RAG = Retrieval-Augmented Generation. Combina busca em base de conhecimento com geração contextualizada pelo LLM.
  • Proposto por Lewis et al. (2020) — NeurIPS, Meta AI Research

Hit Rate@K e Recall@KAula 1 · Avaliação

9/13

Duas métricas de retrieval que respondem perguntas diferentes: achou? e quanto recuperou?

  • Hit Rate@K: proporção de queries com ≥1 doc relevante no top-K — útil para chatbots
  • Recall@K: fração dos relevantes recuperados no top-K — útil para análise jurisprudencial
  • K=1: precisão de ranking (apenas a melhor resposta)
  • K=3-5: cenário típico de RAG (contexto do LLM)
  • K=10: limite superior, útil quando há reranker em seguida
Python · Implementação
def hit_rate_at_k(sims, relev, k):
    n = sims.shape[0]
    hits = 0
    for i in range(n):
        topk = np.argsort(sims[i])[::-1][:k]
        if (relev[i, topk] >= 1).any():
            hits += 1
    return hits / n

# BGE-M3 no corpus jurídico:
# Hit@1 = 0.875 | Hit@3 = 1.000 | Hit@5 = 1.000
# Recall@5 = 0.62 | Recall@10 = 0.91

MRR e NDCG@K — Qualidade do RankingAula 1 · Avaliação

10/13

Hit Rate diz "achou ou não". MRR e NDCG dizem "quão bem ordenou os achados".

  • MRR (Mean Reciprocal Rank) = média de 1/rank do primeiro doc relevante
  • NDCG@K avalia o ranking inteiro com relevância graduada (escala 0/1/2)
  • NDCG normalizado em [0,1]: 1 = ranking ideal, 0.5 = mediano, <0.3 = ruim
  • NDCG é o gold standard em Information Retrieval (TREC, BEIR, MTEB)
  • Para RAG jurídico: priorize NDCG@5 (top-K que vai para o prompt do LLM)

Visualização — UMAP, Histograma e AUC-ROCAula 1 · Avaliação

11/13

Métricas numéricas + visualizações geométricas = decisão informada sobre o modelo.

  • UMAP 2D: reduz embeddings de 768/1024 dims → 2 dims preservando vizinhança
  • UMAP com queries sobrepostas: cada ⭐ deve cair no cluster da sua categoria
  • Histograma Pos vs Neg: distribuições de similaridade — quanto menor a sobreposição, melhor
  • AUC-ROC: probabilidade de um positivo ter sim > um negativo aleatório
  • Benchmark: AUC > 0.85 = excelente para produção · > 0.95 = SOTA
Três Visões Complementares
UMAP 2D
● docs por categoria + ★ queries · linha verde = top-1 correto · linha vermelha = erro
Histograma Pos/Neg
Δμ = média_pos − média_neg · alto Δμ = embeddings discriminativos
Curva ROC
AUC=0.5 aleatório · 0.85 produção · 1.0 perfeito
