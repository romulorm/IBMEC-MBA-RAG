Os 3 Pontos de Falha do Naive RAGAula 3

1/6

O Naive RAG falha por três razões estruturais identificadas na literatura.

  • 1. Query-Document Mismatch: linguagem coloquial vs técnica — perda de 23% de relevância (Ma et al., 2023)
  • 2. Contexto Truncado: chunking fixo corta entidades relevantes entre chunks
  • 3. Sem Filtragem Pós-Retrieval: similaridade semântica ≠ relevância para a pergunta específica
Naive RAG
Query→ Embedding→ kNN→ Top-K→ LLM
Advanced RAG
Query→ Rewrite→ Embedding→ kNN top-20→ Reranker→ top-5→ LLM
✍️

Query Rewriting — Técnica 1: ParaphraseAula 3

2/6

Reformula a query para aproximar o vocabulário do usuário ao vocabulário dos documentos.

  • Usa o próprio LLM para reescrever a query com linguagem técnica-jurídica
  • Exemplo: 'prender alguém sem flagrante' → 'decretação de prisão preventiva art. 311 CPP'
  • Melhora Recall@5 em ~23% em corpus jurídico (Ma et al., 2023)
  • Custo: 1 chamada extra ao LLM por query (~50ms de latência adicional)
Python 3.11+
REWRITE_PROMPT = """
Você é especialista em Direito Brasileiro.
Reescreva a seguinte query usando terminologia jurídica precisa.
Query original: {query}
Query reescrita (apenas o texto, sem explicações):
"""

def rewrite_query(query: str, llm) -> str:
    prompt = REWRITE_PROMPT.format(query=query)
    return llm.generate(prompt, temperature=0.1, max_tokens=100)

# Exemplo
original = "posso ser preso sem ter feito nada?"
rewritten = rewrite_query(original, llm)
# → "prisão preventiva sem flagrante: requisitos legais art. 312 CPP"

Reranking com BGE-RerankerAula 3

3/6

Cross-encoder que re-ordena os top-K resultados do retriever com precisão superior ao bi-encoder.

  • Bi-encoder (retrieval): query e documento são codificados separadamente — rápido, paralelo
  • Cross-encoder (reranking): query e documento são analisados JUNTOS — lento, preciso
  • BGE-Reranker-v2-m3: score de relevância de -∞ a +∞, convertido por sigmoid → [0,1]
  • Pipeline: retrieval top-20 → reranker → top-5 para o LLM
Python 3.11+
from FlagEmbedding import FlagReranker

reranker = FlagReranker(
    "BAAI/bge-reranker-v2-m3",
    use_fp16=True
)

query = "requisitos prisão preventiva"
passages = retriever.get_top_k(query, k=20)

# Reranking: pares (query, passage)
pairs = [[query, p.text] for p in passages]
scores = reranker.compute_score(pairs, normalize=True)

# Reordenar por score decrescente
ranked = sorted(zip(passages, scores),
                key=lambda x: x[1], reverse=True)
top_5 = [p for p, s in ranked[:5]]

Advanced RAG — Pipeline Tri-LayerAula 3

4/6
O Advanced RAG adiciona três camadas de refinamento ao fluxo básico.
  • PRÉ-RETRIEVAL: Query Rewriting → melhora a query antes de buscar
  • RETRIEVAL: busca vetorial no OpenSearch com BGE-M3
  • PÓS-RETRIEVAL: Reranking com BGE-Reranker → seleciona os chunks mais relevantes

RETRIEVAL

PÓS-RETRIEVAL

Modular RAG — Arquitetura DesacopladaAula 3

5/6

O Modular RAG desacopla cada componente do pipeline em módulos intercambiáveis com interfaces contratuais.

  • Módulos: QueryTransformer | Retriever | Reranker | ContextCompressor | Generator
  • Cada módulo tem interface fixa: input → output padronizado
  • Permite trocar componentes sem reescrever o pipeline inteiro
  • Benefício jurídico: testar diferentes retrievers mantendo o gerador constante
Python 3.11+
# Interface contratual do Retriever
from abc import ABC, abstractmethod
from dataclasses import dataclass

@dataclass
class RetrievalResult:
    chunks: list[str]
    scores: list[float]
    metadata: list[dict]

class BaseRetriever(ABC):
    @abstractmethod
    def retrieve(self, query: str, top_k: int = 5) -> RetrievalResult:
        ...

# Implementação OpenSearch (plug-and-play)
class OpenSearchRetriever(BaseRetriever):
    def retrieve(self, query: str, top_k: int = 5) -> RetrievalResult:
        # ... busca no OpenSearch ...
        return RetrievalResult(chunks=..., scores=..., metadata=...)

LangFuse — Observabilidade do PipelineAula 3

6/6

LangFuse instrumenta cada etapa do pipeline com traces e spans para diagnóstico e otimização.

  • Trace: execução completa (uma query do usuário ao final)
  • Span: etapa individual (rewrite, retrieval, reranking, generation)
  • Metrics coletadas: latência por etapa, tokens consumidos, custo estimado
  • Decorator @observe: instrumenta funções Python com uma linha
Python 3.11+
from langfuse.decorators import observe, langfuse_context

@observe()  # trace automático
def pipeline_rag(query: str) -> str:
    # Span: query rewriting
    with langfuse_context.span("query_rewriting") as span:
        rewritten = rewrite_query(query)
        span.update(metadata={"original": query})

    # Span: retrieval
    with langfuse_context.span("retrieval") as span:
        chunks = retriever.retrieve(rewritten, top_k=20)
        span.update(metadata={"n_chunks": len(chunks)})

    # Span: reranking
    with langfuse_context.span("reranking") as span:
        top_chunks = reranker.rerank(chunks, top_k=5)

    # Geração
    return generator.generate(query, top_chunks)
