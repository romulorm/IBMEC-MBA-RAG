FastAPI — Endpoints do Pipeline RAGAula 12

2/5

FastAPI expõe o pipeline RAG como API REST com documentação automática e streaming.

  • POST /query: endpoint principal com streaming de resposta via SSE
  • POST /ingest: ingestão de documentos com validação de tipo e tamanho
  • GET /health: health check para Kubernetes liveness probe
  • GET /metrics: métricas Prometheus (latência, throughput, token count)
Python 3.11+
from fastapi import FastAPI, HTTPException
from fastapi.responses import StreamingResponse
from pydantic import BaseModel

app = FastAPI(title="RAG Jurídico API", version="1.0.0")

class QueryRequest(BaseModel):
    query: str
    top_k: int = 5
    strategy: str = "hybrid"  # naive|advanced|hybrid|agentic

@app.post("/query")
async def query_rag(
    request: QueryRequest,
    token: str = Depends(verify_jwt)
):
    async def generate():
        async for token in pipeline_rag_stream(request.query):
            yield f"data: {json.dumps({'token': token})}\n\n"
    return StreamingResponse(
        generate(), media_type="text/event-stream"
    )

Kubernetes — Deploy SeguroAula 12

3/5

Kubernetes orquestra os componentes do sistema RAG com alta disponibilidade e auto-scaling.

  • Deployment vLLM: 2 replicas com GPU, resource limits, liveness/readiness probes
  • Deployment BGE-M3: 3 replicas CPU/GPU, HPA para escalar em alta demanda
  • OpenSearch StatefulSet: 3 nós persistentes com PVC para dados
  • Secrets: API keys em Kubernetes Secrets, não em variáveis de ambiente plaintext
Python 3.11+
# kubernetes/vllm-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: vllm-llama3
spec:
  replicas: 2
  template:
    spec:
      containers:
      - name: vllm
        image: vllm/vllm-openai:latest
        args:
          - --model=meta-llama/Meta-Llama-3.1-8B-Instruct
          - --gpu-memory-utilization=0.9
        resources:
          limits:
            nvidia.com/gpu: 1
            memory: 24Gi
        livenessProbe:
          httpGet:
            path: /health
            port: 8000
          initialDelaySeconds: 120

Estratégias de SegurançaAula 12

4/5
Sistemas jurídicos exigem proteção rigorosa de dados sensíveis e controle de acesso granular.
  • mTLS: comunicação cifrada entre todos os serviços no cluster
  • RBAC: controle de acesso baseado em papéis (advogado vs delegado vs analista)
  • PII Masking: remoção automática de CPF, RG, dados pessoais antes de indexar
  • Audit Log: registro imutável de cada query e documento acessado (LGPD)
