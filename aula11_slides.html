LLMLingua — Compressão de ContextoAula 11

2/5

LLMLingua remove tokens irrelevantes do contexto, reduzindo custo e latência sem perder qualidade.

  • Token pruning: remove tokens com baixa perplexidade para o contexto da query
  • Compressão 4:1 (75% redução) com mínima perda de qualidade (Microsoft, 2023)
  • Benefício: cabe mais contexto na janela do LLM → menos chamadas de retrieval
  • Custo: 1 passagem extra pelo modelo de compressão (tiny LM)
Python 3.11+
from llmlingua import PromptCompressor

llm_lingua = PromptCompressor(
    model_name="microsoft/llmlingua-2-bert-base-multilingual",
    device_map="auto"
)

# Comprimir chunks recuperados antes de enviar ao LLM
compressed = llm_lingua.compress_prompt(
    context=retrieved_chunks,
    question=query,
    target_token=512,  # de ~2000 para 512 tokens
    rate=0.75          # 75% de compressão
)
print(f"Ratio: {compressed['ratio']:.2f}x")
print(compressed["compressed_prompt"])

ColBERT — Late InteractionAula 11

3/5

ColBERT gera um embedding por token (não por sentença), permitindo busca token-a-token com eficiência.

  • Bi-encoder clássico: 1 vetor por documento — rápido, menos preciso
  • Cross-encoder: atende tokens juntos — preciso, muito lento para retrieval
  • ColBERT late interaction: 1 vetor por token, score = MaxSim(Q, D) — preciso e escalável
  • RAGatouille: biblioteca Python que simplifica ColBERT (fine-tuning + indexação + busca)
Python 3.11+
from ragatouille import RAGPretrainedModel

# Indexar corpus com ColBERT
RAG = RAGPretrainedModel.from_pretrained("colbert-ir/colbertv2.0")

RAG.index(
    collection=corpus_juridico,
    index_name="juridico_colbert",
    max_document_length=512,
    split_documents=True
)

# Busca token-a-token
results = RAG.search(
    query="requisitos prisão preventiva art 312 CPP",
    k=5
)
for r in results:
    print(f"Score: {r['score']:.3f} | {r['content'][:150]}")

Time-Aware RAGAula 11

4/5

Aplica decay temporal para priorizar documentos mais recentes em queries sobre legislação e jurisprudência.

  • Problema: acórdão de 2015 e de 2024 têm mesmo score de similaridade, mas o recente é mais válido
  • Decay function: score_final = score_semântico × e^(-λ × anos_desde_publicação)
  • λ (lambda): controla velocidade do decay — maior λ = decai mais rápido
  • OpenSearch: custom scoring script combina kNN score + temporal decay

DSPy — Otimização AutomáticaAula 11

5/5
DSPy otimiza automaticamente os prompts do pipeline RAG via programação declarativa.
  • Define o pipeline como programa declarativo (módulos com assinaturas de entrada/saída)
  • Otimizador (BootstrapFewShot): gera e testa exemplos few-shot automaticamente
  • Compila: encontra os melhores prompts sem engenharia manual
  • Resultado: pipelines mais robustos com menos trabalho de prompt engineering
