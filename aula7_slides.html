O Vocabulary Mismatch ProblemAula 7

1/5

A maior barreira do RAG jurídico: o usuário fala coloquial, os documentos falam técnico.

  • 'demitir funcionário' → documentos usam 'rescisão contratual' / 'dispensa sem justa causa'
  • 'pagar para sair da cadeia' → documentos usam 'liberdade provisória mediante fiança'
  • 'bater na esposa' → documentos usam 'lesão corporal no âmbito doméstico / violência de gênero'
  • Resultado: queries coloquiais têm recall 23% menor que queries técnicas equivalentes
  • Solução: Query Enhancement — gerar múltiplas perspectivas da mesma intenção

Multi-Query RAGAula 7

2/5

Gera N variações da query original com diferentes perspectivas linguísticas e técnicas.

  • LLM gera N=3-5 variações: coloquial → técnico, sinônimos, ângulos diferentes
  • Executa N buscas paralelas no OpenSearch
  • Deduplicação: remove chunks idênticos ou muito similares (coseno > 0.95)
  • União dos resultados: maior recall a custo de N×latência de retrieval
Python 3.11+
from langchain.retrievers import MultiQueryRetriever
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    base_url="http://localhost:8000/v1",
    model="llama-3.1-8b-instruct",
    temperature=0.1
)

retriever = MultiQueryRetriever.from_llm(
    retriever=opensearch_retriever,
    llm=llm,
    prompt=JURIDICO_MULTIQUERY_PROMPT
)

# LangChain gera variações automaticamente
docs = retriever.get_relevant_documents(
    "posso ser preso sem ter feito nada?"
)
print(f"Chunks únicos recuperados: {len(docs)}")

Step-Back PromptingAula 7

3/5

Abstrai a query específica para um nível mais geral que tem maior cobertura nos documentos.

  • Query específica: 'Posso ficar em silêncio na delegacia quando o delegado me interrogar?'
  • Step-Back: 'Quais são os direitos constitucionais do investigado durante o inquérito policial?'
  • A query abstraída cobre mais documentos e traz contexto jurídico mais rico
  • Quando NÃO usar: queries já técnicas e específicas — abstração pode perder precisão
Python 3.11+
STEP_BACK_PROMPT = """
Você é especialista em Direito Brasileiro.
Dada uma pergunta específica, formule uma versão mais GERAL
que capturaria os princípios jurídicos fundamentais relacionados.

Pergunta específica: {query}
Pergunta geral (step-back):
"""

def step_back_retrieve(query: str, llm, retriever):
    # Gerar query abstraída
    abstract_query = llm.generate(
        STEP_BACK_PROMPT.format(query=query),
        temperature=0.1, max_tokens=100
    )

    # Retrieval duplo
    specific_docs = retriever.retrieve(query, top_k=3)
    abstract_docs = retriever.retrieve(abstract_query, top_k=3)

    # Combinar (sem duplicatas)
    return deduplicate(specific_docs + abstract_docs)

RAG-Fusion — Fusão por RRFAula 7

4/5
RAG-Fusion une os rankings de múltiplas queries via Reciprocal Rank Fusion para máximo recall.
  • Gera N sub-queries via LLM (decomposição da query original)
  • Executa N buscas PARALELAS (asyncio) para mínima latência
  • Combina rankings com RRF: RRF(d) = Σ 1/(k + rank_i(d))
  • Resultado: lista fusionada com mais recall e melhor ordenação que qualquer busca individual
Python 3.11+
import asyncio
from langchain.load import dumps, loads

async def parallel_retrieval(queries: list[str]) -> list:
    """Executa N buscas em paralelo com asyncio"""
    tasks = [retriever.aget_relevant_documents(q)
             for q in queries]
    results = await asyncio.gather(*tasks)
    return results

def reciprocal_rank_fusion(results: list, k: int = 60):
    scores = {}
    for docs in results:
        for rank, doc in enumerate(docs):
            key = dumps(doc)  # serializa para hashable
            scores[key] = scores.get(key, 0) + 1/(rank + k)
    return [
        loads(doc) for doc, _ in
        sorted(scores.items(), key=lambda x: x[1], reverse=True)
    ]

Trade-off: Recall × Latência × CustoAula 7

5/5

N queries gera N × custo — a escolha do N depende do cenário de uso.

  • N=1 (baseline): latência ~200ms, custo 1×, recall base
  • N=3 (Multi-Query): latência ~350ms, custo ~2.5×, recall +18%
  • N=5 (RAG-Fusion): latência ~500ms, custo ~4×, recall +27%
  • Recomendação: N=3 para produção geral, N=5 apenas para consultas críticas de alta precisão
